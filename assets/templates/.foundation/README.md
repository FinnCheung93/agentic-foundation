这里是本项目的 agentic 治理地基。它不替代需求文档、执行方案或项目计划；它负责让后续 agent 的工作有原则、有状态、有记录、有检查、有待裁决建议。

## 文件职责

| 文件 | 作用 | 更新时机 |
|---|---|---|
| `AGENTS.md` | agent 工作入口，定义 Mandatory Loop、Hard Rules 和冲突处理 | 入口协议、工作循环或硬规则需要调整时 |
| `PRINCIPLES.md` | 长期原则、禁区和取舍依据 | 信息具备长期约束意义且裁决清晰时 |
| `STATE.md` | 当前接续快照，回答“现在接手要知道什么” | 当前状态、焦点、阻塞或接续点变化时 |
| `LOG.md` | 历史账本，记录重要事件 | 发生重要变更、决策、事故、验收或复盘时 |
| `CHECKS.md` | 检查清单，用于执行前、执行中、完成后自检 | 检查项、验证习惯或误投扫描需要调整时 |
| `SUGGESTIONS.md` | 未裁决治理提案队列，覆盖 `AGENTS.md` 和 `.foundation/` 下任一治理文件 | AI 自发发现治理文件需要演进但尚未获得裁决时 |

## 记忆路由表

本文档是唯一记忆路由权威。其它治理文件可以引用或检查本表，不应重新定义路由。

| 信息性质 | 倾向写入 |
|---|---|
| 入口协议、工作循环、硬规则、冲突处理 | `AGENTS.md` |
| 长期约束、禁区、稳定偏好、取舍依据 | `PRINCIPLES.md` |
| 当前恢复所需状态、工作焦点、阻塞、接续点 | `STATE.md` |
| 已发生的重要事件、决策、事故、验收、复盘 | `LOG.md` |
| 执行前后检查项、验证习惯、误投扫描 | `CHECKS.md` |
| AI 自发发现但尚未裁决的治理改进 | `SUGGESTIONS.md` |

记录前先判断信息性质。不确定该写到哪里时，先说明判断并请用户裁决；不要把所有信息都塞进同一个文件。

## 扩展项目文档

项目可以按需维护架构、领域、开发约定、决策、需求、操作手册、Agent 专用适配等扩展文档；这些文档不由本地基默认创建。新增后应在下表登记职责、依赖性质和读取时机，避免散落或与通用入口冲突。

| 文档 / 目录 | 职责 | 依赖性质 | 何时读取 |
|---|---|---|---|
| 待补充 | 待补充 | 硬依赖 / 软依赖 | 待补充 |

扩展规则：

- `AGENTS.md` 只做短入口和索引，不承载完整业务、架构、开发规范或操作手册。
- 硬依赖是缺失会导致任务错误、越权或无法验证的资料；缺失时先对齐。软依赖是有助于提高质量但不阻塞执行的资料；缺失时继续工作，必要时记录改进建议。
- 优先维护通用入口：`README.md`、`AGENTS.md`、`.foundation/`。`.cursor/`、`.codex/`、`.claude/` 等专用目录只作为薄适配层，不应形成另一套冲突规则。
- 团队级、语言级、平台级的大型规范不在项目内全文复制；项目内只保留摘要、链接和本项目例外。
- Agent 或 skill 生成的专用目录不能替代长期基础文档；只有任务明确需要时才读取。
- 共享语言文档可以作为软依赖登记，用于统一项目术语和避免混用；它不承载 spec、实现细节或计划。
- 重要长期决策按现有路由沉淀到 `LOG.md`、`PRINCIPLES.md` 或 `SUGGESTIONS.md`；不默认创建 ADR 文件夹。
- Foundation 只负责登记、路由和检查，不接管 PRD、Specs、任务拆分、架构设计或其它专业流程。

## 任务读取策略

为控制上下文成本，每轮不要默认读取所有文档。先判断任务类型，再读取相关资料：

| 任务类型 | 建议读取 |
|---|---|
| 普通小改 / 文案 / 局部整理 | `AGENTS.md`、本文件、`STATE.md`，按需读取检查清单 |
| 涉及长期原则、禁区、稳定偏好 | `PRINCIPLES.md` |
| 涉及当前进度、阻塞、恢复入口 | `STATE.md` |
| 涉及历史决策、事故、验收、复盘 | `LOG.md` |
| 涉及验证、发布、回退、主交付物 | `CHECKS.md` 和已登记的操作手册 |
| 涉及架构、领域、开发约定、API、需求或运行手册 | 读取上方已登记的对应扩展文档 |
| 涉及 PRD、Specs、任务拆分、架构设计或专业执行流程 | 读取已登记资料后交给对应专业流程；foundation 只保留治理记录 |

## 读取规则

- 每轮开工前读：`AGENTS.md`、本文件、`PRINCIPLES.md`、`STATE.md`。
- 重大改动前读：`CHECKS.md`。
- 发生变更、事故、迭代或复盘后，按记忆路由表判断是否记录。
