# Checks

本文档是重要工作前后的检查表。它只负责检查，不重新定义文件职责或记忆路由；文件职责和路由以 `.foundation/README.md` 为准。

## 执行前检查

- 已读取 `AGENTS.md`、`.foundation/README.md`、`.foundation/PRINCIPLES.md`、`.foundation/STATE.md`。
- 已确认本轮任务类型：治理 / PRD / Specs / 实现 / 其它。
- 已按任务类型读取必要项目资料，没有无差别全量读取全部文档。
- 已明确本轮目标、边界、影响范围和验收方式。
- 重要改动已明确可运行、可观察、可重复的反馈环；暂时没有反馈环时，已说明风险和替代验证方式。
- 已说明会触碰哪些文件或模块、依赖哪些文件或模块、不触碰哪些文件或模块。
- 已确认是否存在冲突文件、旧结构或用户未裁决事项。

## 执行中验证

- 优先使用脚本、临时目录、最小样例、预览或局部路径验证。
- 不能旁路验证时，说明原因和风险。
- 临时原型、样例、预览或脚本已说明要回答的问题，不把临时产物当作长期结构。
- 发现模块边界不清时，先暂停并请求裁决。
- 不为了验证方便扩大改动范围。

## 执行后检查

- 已运行可用检查、预览或最小验证。
- 已根据验证结果修复问题并再次验证。
- 临时验证产物已有归宿：删除、吸收、记录结论，或转为待裁决建议。
- 已检查本轮是否触发治理钩子；如触发，已按信息性质记录、更新或提出待裁决建议。
- 已判断本轮产生的信息性质，并按 `.foundation/README.md` 的唯一记忆路由记录。
- 已更新当前接手快照，或确认 `STATE.md` 仍能反映当前目标、状态、下一步、约束、验收方式和范围外事项。
- 已检查是否把已有 artifact 全文复制进 `STATE.md` 或 `LOG.md`；如有，已改为引用路径、文件名或简短摘要。
- 已确认本轮下一步、验收方式和范围外事项足够清楚，下一位 agent 能接手。
- 已检查是否产生长期原则、禁区、稳定偏好或取舍依据；若被误写进 `STATE.md` 或 `LOG.md`，已补写或迁移到更合适的治理文件。
- 已确认重要事件、决策、事故、验收或复盘记录在 `LOG.md`。
- 已检查本轮是否触及已裁决的拒绝、不做或暂不支持方向；如触及，已引用或补充对应记录。
- 已检查重要决策是否难回退、未来无上下文会疑惑、且来自真实取舍；如是，已按现有路由沉淀，不默认创建 ADR 目录。
- 已检查是否把易变路径、代码片段或临时实现细节误写入长期治理文件；如有，已迁移到更合适的位置或标注有效范围。
- 已确认 AI 自发发现但尚未裁决的治理建议进入 `SUGGESTIONS.md`。
- 已快速查看 `SUGGESTIONS.md` 中待确认提案；若影响本轮工作，已请求裁决或处理状态。
- 已检查本轮是否出现合格治理信号；若出现，已更新 `STATE.md` 的治理脉冲。
- 若治理脉冲达到 3 / 3，已在自然停顿点完成轻量复盘，或记录待复盘信号且未开启新阶段。
- 已确认记录位置符合 `.foundation/README.md` 的唯一记忆路由。
- 已检查本轮是否暴露治理文件职责、触发条件或检查项需要演进；未裁决时进入 `SUGGESTIONS.md`。

## 发布一致性检查

当 Skill 本体、模板、脚本或生成物能力发生变化时：

- 已判断是否需要同步 Skill 版本、Foundation 版本、脚本常量和生成物版本标记。
- 已判断是否需要更新验证记录、项目地基状态或历史日志。
- 已用临时目录、静态检查或其它可复现方式验证未来生成物。
- 不把未同步版本或未记录验证的本体变更称为完成。

## 生成物职责去重检查

- `AGENTS.md` 只包含入口协议、Mandatory Loop 和 Hard Rules。
- `.foundation/README.md` 是唯一文件职责与记忆路由权威。
- `.foundation/README.md` 已登记扩展项目文档的职责和读取时机；未登记文档不应变成隐藏入口。
- `.foundation/README.md` 已区分扩展文档的硬依赖 / 软依赖，且未让 foundation 接管专业流程。
- `.foundation/PRINCIPLES.md` 只包含长期原则、禁区和取舍依据。
- `.foundation/STATE.md` 聚焦当前恢复所需状态、工作焦点、阻塞和接续点。
- `.foundation/STATE.md` 通过修订保持当前接手快照，包含下一步、约束、验收方式和范围外事项；最近接续点只保留 3-5 条，并维护治理脉冲。
- `.foundation/LOG.md` 聚焦重要事件、决策、变更、事故、验收和复盘。
- `.foundation/CHECKS.md` 只做检查表，不重新定义路由。
- `.foundation/SUGGESTIONS.md` 只记录未裁决治理提案。
